## Overview

This **Anonymous Reporting System** allows users to report problems or incidents without revealing their identity.  
It is designed for environments like **schools, companies, or communities** where privacy is crucial, and no registration is required.

## Key Features

- **Anonymous Reporting**: Users can submit reports without creating an account.  
- **Public Access & Visibility**: Reports can be viewed publicly while keeping the reporter anonymous.  
- **Moderation & Management**: Admin interface to manage, track, and respond to reports.  
- **Real-time Notifications**: Optional alerts to admins when a new report is submitted.  
- **Security & Privacy**: No personal data is stored; all submissions are encrypted.  

## Technologies Used

- **Elysia.js (Bun)** – backend framework for fast API handling.  
- **TypeScript** – for type-safe code and maintainability.  
- **Prisma + PostgreSQL** – database to manage reports securely.  
- **Tailwind CSS & Next.js** – for a responsive and user-friendly frontend.  

## Challenges and Learnings

The main challenge was **ensuring true anonymity** while still giving admins enough data to act on reports.  
It required balancing **security, performance, and usability** without compromising privacy.

## Outcome

The system is now a robust tool for anonymous reporting, helping institutions **detect and solve issues effectively** while protecting user identity.  
It demonstrates expertise in **modern backend development, privacy-first design, and full-stack architecture**.
